Some aesthetic changes I would recommend. Not all are deal breakers, so if Claude is having difficulty executing or understanding some of these, we can edit down.

* **Change the tracker’s top navigation background to ESP cream**  
   Set the header and navigation background to `#FFFDF2` so it matches the header treatment used across the main ESP website.  
* **Use Public Sans throughout, with one deliberate serif exception**  
  Use Public Sans for all interface text, labels, buttons, headings, chart text, and supporting copy. Use Publico Banner only for large numbers that serve as major points of emphasis, such as the figures in the “Hours of Work” summary.  
* **Remove the visible section-number labels**  
  Remove the `§01` and `§02` labels entirely. The layout and headings should establish the section hierarchy without visible numbering.  
* **Simplify the metric-card borders**  
  Remove the bright full-width colored line across the top of each card. Use a consistent neutral border around every card, then retain category color only in a small label, icon, or short accent mark. This will make the dashboard feel like one ESP product rather than several unrelated chart styles.  
* **Begin chart colors with the established ESP palette**  
  Assign the first chart series the following colors in order: ESP navy \#2c3254, ESP green \#70ad8f, and ESP gold \#ebc382. Additional series can introduce new colors, but they should remain within the same muted, warm visual range rather than shifting to highly saturated or neon colors.  
* **Replace the small “Show” triangles with single chevrons**  
  Remove the text-based triangle that appears beside “Show” on the top right menu stack. Use a simple single navy downward chevron with the same stroke weight as the arrow beside “Read More” elsewhere on the ESP site. Place the chevron to the right of “Show,” leave clear space between the word and icon, and rotate it upward when the section is expanded.  
* **Use the same rectangular button shape throughout the tracker**  
   Replace fully rounded pills with lightly rounded rectangles. Use a consistent corner radius of about `2–4px`, a `1px` navy border, and more horizontal padding. This should apply to tabs, date controls, view toggles, and download buttons.  
* **Create one clear active and inactive treatment**  
   Active buttons should use ESP navy `#2c3254` with white text. Inactive buttons should use the surrounding cream background with navy text and a thin navy border. Avoid using different accent colors to indicate ordinary interface states.
