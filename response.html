Responses to the internal team's tracker feedback

Eight requests from edit_questions.md, each with a recommendation, an effort estimate, and a concrete 1–2–3 path. Read in the context that matters: the reviewer sees the tracker embedded in the ESP site (?embed=1), where the navy hero and topbar are hidden, the view tabs sit on a light bar, and backgrounds go white.

One thing shapes almost everything below. The tracker was built with near-ESP values, not exact ones. Its navy is #1B2A4E (ESP navy is #2c3254), its paper is #FBF7EE (ESP cream is #FFFDF2), and its type uses Google substitutes (Source Serif 4 for Publico, Inter Tight for Neue Haas) rather than the licensed ESP fonts. In embed mode those near-misses sit directly against the real ESP page chrome, which is exactly where the eye catches the mismatch. So the highest-leverage move is a Step 0: retokenize — repoint the NAVY/PAPER/palette constants at ESP's true values and embed the real fonts. Items 1, 7, and 8 then largely fall out for free. Decided: we are doing the full retokenize as Step 0.
Recommend Do, with a caveat Confirm scope first
1

Top navigation background → ESP cream #FFFDF2

RecommendEffort: Low
Verdict: Do it. In embed the "top navigation" is unambiguous — it's the light view-tabs/controls bar (there is no navy hero in the iframe). Matching it to the ESP site header's cream is a clean win and removes a visible seam between the iframe and the page around it.
  1. Change the embed menu background from #F4F2E4 to #FFFDF2 (the MENU_BG constant / embed branch).
  2. Set the page/chart surface behind the tabs to the same cream so the bar doesn't float on stark white.
  3. Re-check tab contrast on cream in embed_test.html at mobile and desktop widths.
Note: standalone mode still has the navy hero; this change is embed-scoped, so we won't disturb the standalone look.
2

Public Sans throughout, Publico Banner only for big emphasis numbers

Do, with a caveatEffort: High
Verdict: Strongly on-brand and worth doing, but it's the biggest single change and carries a real tension. The tracker leans on serif in ~40 places (card titles, captions, headline figures). Paring serif back to only large emphasis numbers is a genuine character shift, not a tweak.
  1. Embed the licensed fonts via @font-face (Public Sans + Publico Banner Medium from the ESP style-guide bundle) and drop the Google substitute links.
  2. Reassign the ~40 fontFamily: 'Publico…' declarations to Public Sans, keeping Publico Banner only on the big figures (e.g. the Hours-of-Work stats).
  3. Fold the Neue Haas Grotesk layer (eyebrows, labels, tabs, axis text) down to Public Sans too, then sweep every view for stragglers.
Decided — follow the note literally. Public Sans for everything — labels, headings, chart text, body — with Publico Banner reserved only for the large emphasis figures. This also means the current Neue Haas Grotesk substitute (Inter Tight), used today on eyebrows, labels, tabs, and axis text, gets replaced by Public Sans rather than the real Neue Haas. Step 2 of the item above therefore folds both serif and the Neue Haas layer down to Public Sans.

4

Simplify metric-card borders — drop the colored top rule

RecommendEffort: Medium
Verdict: Agree, and this is the single biggest reason the dashboard reads as several unrelated widgets. The per-card colored top rule is doing the damage; category color already survives in the card's small eyebrow label, so we keep the association without the noise.
  1. Replace the four borderTop: 2px solid {accent} rules with one neutral 1px border around the whole card.
  2. Keep category color only in the small eyebrow label (or a short accent tick).
  3. Reconcile with the existing grid dividers (border-right) so borders don't double up.
5

Chart colors start with ESP navy / green / gold

Do, with a caveatEffort: High
Verdict: Right instinct, hardest item. Series colors are baked into the data payloads by fetch_data.R in per-category families (debt=red, labor=green, etc.). Re-running the R pipeline needs API keys and a fresh multi-minute pull; a front-end color override is cleaner, reversible, and needs no data refresh.
  1. Add an ordered ESP palette constant — #2c3254, #70ad8f, #ebc382, then muted warm extensions — in index.html.
  2. Override item.color and PIN_COLORS at render from that palette instead of the baked hexes.
  3. Tune single-series national cards into the muted range; order navy/green/gold in the multi-line Compare and pinned-state views.
Caveat: today's category colors encode meaning (a color family per category). Flattening everything to navy/green/gold first weakens that coding. We can preserve the semantics by recoloring each family to a muted ESP tone instead of a strict global order — the two goals partly conflict, so we'll pick a balance.
6

Replace "Show" triangles with a single navy chevron

RecommendEffort: Low
Verdict: Fine, minor polish. Two spots use text triangles (▾ Show). An inline SVG chevron matches the "Read More" affordance on the ESP site and rotates cleanly.
  1. Swap the text triangles for a small inline SVG chevron, navy, ~1.75px stroke to match the ESP "Read More" arrow.
  2. Leave clear space between the word "Show" and the icon.
  3. Rotate 180° on expand via a CSS transform.
7

One rectangular button shape throughout (2–4px radius, 1px navy border)

RecommendEffort: Medium
Verdict: Good consistency win. Today it's mixed: anchor buttons and category pills are full pills (borderRadius: 100), chips are 6px, view tabs have none. Mechanical, but many touch points.
  1. Set a shared ~3px radius and 1px navy border across tabs, anchor/date controls, view toggles, category pills, card range buttons, and CSV/PNG buttons.
  2. Add uniform horizontal padding.
  3. Sweep each view (National / State / Compare / Map) for stragglers.
8

One clear active / inactive treatment

RecommendEffort: Medium
Verdict: Agree. Most controls are already navy-active / white-text. The real outlier is the metric chip, which tints the category accent for its selected state — exactly the "different accent colors for ordinary states" the note wants gone.
  1. Standardize every control: active = navy #2c3254 + white; inactive = cream + navy text + 1px navy border.
  2. Move the chip's selected state off the category accent onto the navy standard.
  3. Decide whether any chip→line color link survives (e.g. keep a small color dot on the chip).
Caveat: the chip's accent tint currently ties a chip to its chart-line color. Removing it loses that visual link — acceptable, but worth naming so it's a choice, not an accident.

Suggested sequence

Step 0 retokenize to true ESP values + embed real fonts → then the cohesion pass (1, 3, 4, 6, 7, 8) which is mostly mechanical → then the two heavy, decision-bearing items (2 typography and 5 chart palette) last. Effort at a glance: 3 and 6 are quick; 1, 4, 7, 8 are moderate; 2 and 5 are the lifts.

Working draft — both decisions are settled: full retokenize as Step 0, and Public Sans literally everywhere except large emphasis numbers. This page is styled in the true ESP palette (navy #2c3254, cream #FFFDF2, green #70ad8f, gold #ebc382) as a live sample of the target.
